简介与核心理念

为什么选择 Letta Code？

📋

传统会话式工具

大多数 AI 编码工具是基于会话的，每次打开新窗口，AI 都是一张白纸。上下文仅限于当前对话历史 + AGENTS.md 等静态文件。

👤
就像每次都雇用一个新的外包合同工。
❌ 无记忆 ❌ 无成长

Letta Code

推荐

Letta Code 是基于 Agent 的，具有以下特点：

  • 持久化：同一个 Agent 可以跨会话、跨天、跨月使用。
  • 成长性：Agent 会将重要的上下文写入其长期记忆。
  • 模型自由：可以随意切换底层模型，而不会丢失记忆。
🤖
就像拥有一个能够不断学习、记住你习惯的长期合作伙伴或徒弟。
✅ 持久记忆 ✅ 持续成长
功能

核心功能

强大的功能，极致的体验

💾

持久化记忆

Letta Code 的核心是 Memory Blocks（记忆块）。Agent 拥有专门的存储区域来保存：

  • User Persona: 关于用户的偏好
  • Project Context: 项目的架构、依赖、构建方式

使用 /clear 命令可以清空当前的对话缓冲区，但不会删除 Agent 的长期记忆。

$ /clear
🔄

模型无关性

Letta Code 不绑定特定模型。你可以随时切换底层 LLM，而 Agent 的记忆和状态保持不变。

$ /model
$ /model gpt-4o
$ /model claude-3-5-sonnet
📚

技能学习

Agent 可以通过 .skills 目录加载 Python 编写的可重用技能函数。更重要的是，你可以让 Agent 从刚才的操作中学习新技能。

$ /skill 将刚才重构代码的逻辑保存为一个名为 "refactor_auth" 的技能
🤖

子 Agent

为了处理复杂任务，主 Agent 可以生成专门的"子 Agent"来并发处理工作：

  • explore: 专门用于只读搜索代码库，速度快
  • plan: 用于规划复杂任务
  • general-purpose: 具备完整能力的通用子 Agent

开始

快速开始

三步上手，即刻体验

1

启动 Letta Code

1 分钟

导航到你的项目目录并运行：

$ cd your-project-directory
$ letta

如果是首次运行，系统会提示你通过浏览器进行 OAuth 登录。登录成功后，Letta Code 会自动为你创建一个针对当前目录的 Agent。

2

初始化记忆 (/init)

2 分钟

首次进入项目时，建议运行初始化命令。这会让 Agent 主动扫描项目结构、读取 README 和配置文件。

$ /init

Agent 会根据扫描结果更新其"核心记忆（Core Memory）"，记录项目概况和常用命令。

3

教导 Agent (/remember)

30 秒

当 Agent 犯错或你需要告知它重要信息时，使用 /remember 命令将其写入长期记忆。

$ /remember 请在本项目中始终使用 pnpm 而不是 npm

下次会话中，Agent 依然会记得这条规则。

命令

命令参考手册

完整的命令列表与说明

命令 描述 类别
/init 初始化。扫描当前目录，建立项目认知的初始记忆。 初始化
/remember [text] 记忆。强制将指定文本写入 Agent 的长期核心记忆中。 记忆
/model 切换模型。打开选择器切换 LLM (如 GPT-4, Claude 3.5)。 配置
/toolset 切换工具集。在不同的预设工具配置间切换 (如 default, codex)。 配置
/clear 清理上下文。清空当前对话历史，但保留长期记忆。 管理
/pin [name] 置顶/保存。将当前 Agent 保存并命名，以便全局或其他项目复用。 管理
/unpin 取消置顶。移除当前 Agent 的置顶状态。 管理
/pinned 查看置顶。列出所有已保存的 Agent。 管理
/skill [instruction] 学习技能。指示 Agent 根据当前上下文学习或创建新技能。 技能
/exit 或 /quit 退出 Letta Code。 系统

CLI 启动参数

在终端启动 letta 时可用的参数：

$ letta [options]
参数 描述
--new 强制创建一个新的 Agent
--agent <id> 连接到指定 ID 的现有 Agent
--model <model> 启动时指定使用的模型
-p, --prompt <text> 无头模式，非交互式运行
--debug 启用调试模式
--version 显示版本信息
🚀

无头模式 (Headless Mode)

Letta Code 支持非交互式运行，非常适合集成到 CI/CD 管道或脚本中。

$ letta -p "Review the code in this directory and write a README.md"

你也可以结合 --yolo 参数（需谨慎），跳过工具执行的确认步骤，实现完全自动化。

🔗

连接自托管服务器

默认情况下，Letta Code 连接到 Letta 官方云平台。如果你自己部署了 Letta Server，可以通过环境变量配置连接：

$ export LETTA_BASE_URL="http://localhost:8283"
$ letta
⚙️

配置文件

Letta Code 会在项目根目录和用户主目录查找配置：

  • .letta/settings.json: 项目级配置（建议提交到 Git）
  • .letta/settings.local.json: 本地覆盖配置（建议加入 .gitignore）
{
  "model": "claude-3-5-sonnet-20240620",
  "contextWindow": 128000
}
